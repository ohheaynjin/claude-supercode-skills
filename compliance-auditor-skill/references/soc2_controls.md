# SOC 2 제어 참조

## 개요
SOC 2(시스템 및 조직 제어) 보안, 가용성, 처리 무결성, 기밀성 및 개인 정보 보호에 대한 유형 2 검사 기준입니다.

## 신뢰 서비스 기준(TSC)

### 공통 기준

1. **CC1.1 - 제어 환경**
   거버넌스 및 위험 관리 프로세스를 확립해야 합니다.

2. **CC2.1 - 책임 전달**
   책임은 조직 전체에 전달되어야 합니다.

3. **CC3.1 - 위험 평가**
   조직은 위험을 식별하고 평가해야 합니다.

4. **CC4.1 - 시스템 모니터링**
   보안 이벤트를 감지하려면 시스템을 모니터링해야 합니다.

5. **CC5.1 - 시스템 제어**
   컨트롤을 설계하고 구현해야 합니다.

6. **CC6.1 - 시스템 유지 관리**
   지속적인 운영을 위해서는 시스템을 유지관리해야 합니다.

7. **CC7.1 - 시스템 데이터**
   데이터는 수명주기 전반에 걸쳐 보호되어야 합니다.

8. **CC8.1 - 공급업체 관리**
   제3자 관계를 관리해야 합니다.

## 보안 기준(CC6)

### CC6.1 - 논리적 및 물리적 액세스 제어

**CC6.1**
- 시스템 구성요소에 대한 논리적 접근은 제한되어야 합니다.
- 최소 권한을 기준으로 접근 권한 부여
- 승인된 담당자에 의해 접근이 승인되었습니다.
- 주기적인 접속 검토

**구현 체크리스트:**
- [ ] 고유 식별자로 생성된 사용자 계정
- [ ] 비밀번호 복잡성 요구 사항 시행
- [ ] 다단계 인증 구현
- [ ] 액세스 승인 워크플로가 마련되어 있습니다.
- [ ] 직무에 따라 액세스 권한이 부여됨
- [ ] 접근 검토 일정 수립(분기별)
- [ ] 해지 시 즉시 철회

**증거:**
- 사용자 액세스 요청 양식
- 접근 승인 문서
- 접속 검토 보고서
- MFA 구성 문서

### CC6.2 - 논리적 액세스 보안

**CC6.2**
- 논리적 접근 보안시설을 관리해야 함
- 물리적, 환경적 접근 보호
- 의심스러운 활동에 대한 논리적 액세스 모니터

**구현 체크리스트:**
- [ ] 로그인 실패 모니터링이 구성되었습니다.
- [ ] 시도 실패 후 계정 잠금
- [ ] 세션 시간 초과가 구현되었습니다.
- [ ] 동시 로그인 제한
- [ ] 지리적 제한이 구성됨
- [ ] 의심스러운 활동 경고가 구성되었습니다.

**증거:**
- 보안 구성 파일
- 모니터링 로그
- 경고 구성 문서

### CC6.3 - 시스템 경계

**CC6.3**
- 시스템 경계를 식별해야 합니다.
- 경계에서의 접근 통제
- 네트워크 분할 구현

**구현 체크리스트:**
- [ ] 시스템 아키텍처 문서화
- [ ] 네트워크 세그먼트가 정의됨
- [ ] 방화벽 규칙이 문서화되었습니다.
- [ ] 공개 서비스를 위해 구성된 DMZ
- [ ] 내부 네트워크 분할
- [ ] 각 경계의 접근 통제

**증거:**
- 네트워크 다이어그램
- 방화벽 규칙 세트
- 시스템 아키텍처 문서
- 경계 문서

### CC6.4 - 암호화

**CC6.4**
- 전송 중인 데이터에 대한 암호화 구현
- 미사용 데이터에 대한 암호화 구현
- 주요 관리 절차

**구현 체크리스트:**
- [ ] 모든 연결에 대한 TLS 1.2+
- [ ] 원격 액세스용 VPN
- [ ] 데이터베이스 암호화(AES-256)
- [ ] 파일 시스템 암호화
- [ ] 백업 암호화
- [ ] 키 순환 일정
- [ ] 안전한 키 저장

**증거:**
- SSL/TLS 인증서
- 암호화 구성 파일
- 주요 관리 절차
- 암호화 정책 문서

### CC6.5 - 시스템 구성 요소 모니터링

**CC6.5**
- 보안 이벤트를 모니터링하는 시스템 구성 요소
- 이상활동 탐지
- 보안 이벤트 상관관계

**구현 체크리스트:**
- [ ] 보안 정보 및 이벤트 관리(SIEM)
- [ ] 모든 시스템에서 로그 수집
- [ ] 실시간 모니터링
- [ ] 경고 임계값이 구성되었습니다.

**증거:**
